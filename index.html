The Problem

Small businesses run financially blind.

All the data exists. It just never comes together in one place — until it's too late.

01
⛓️
Data everywhere, connected nowhere

Bank, POS, payroll, invoices, vendor statements — each in a separate tool. None of them talk to each other automatically. The bookkeeper manually reconciles them every month.

02
📄
Documents buried in Telegram

Clients send PDFs and invoices via chat. They get buried, forgotten, and unprocessed for weeks. No one knows what's been handled or what's still pending.

03
🔍
Errors discovered at month-end

The owner finds out something went wrong during close — not when the transaction happened, when it was still easy to fix. By then, weeks of decisions were made on wrong data.

04
🧠
Everything depends on one expert

All the reconciliation logic lives in one person's head. That is not a system — it is a single point of failure. When they're out, the whole process stops.

How It Works

Every financial event triggers a cycle.

The agent observes, reasons, decides, and acts — then starts again. No manual reconciliation required.

1
Event arrives

A bank transaction, POS report, Telegram PDF, or payroll run enters the system in real time. Any financial event triggers the agent immediately.

2
Agent reads context

FCore Agent checks QuickBooks, bank feed, POS history, and open items. It builds a complete picture across all connected systems before making any decision.

3
Scores confidence

The agent assesses how certain it is about the correct action. High confidence means it acts. Lower confidence means it asks. Ambiguity means it flags for human review.

4
Acts, asks, or alerts

Action is logged. Bank account verified. QuickBooks updated. The loop closes — and the cycle starts again on the next event, continuously.

≥85% confident
Autonomous

The agent executes the correct action in QuickBooks, logs it, and reports in a daily summary. The operator sees what was done — not a question to answer.

Bank deposit matches POS sales → revenue entry posted, bank reconciled.
55–84% confident
Collaborative

The agent stops and sends a structured question via Telegram: the facts, the gap, and 2–3 labeled options. The operator replies with a letter. The agent executes.

Payment $3,200 received, open invoice is $3,000 → options A, B, C presented.
<55% confident
Alert

The agent flags the situation without attempting any action. Raw facts and what is missing are presented clearly. A full human decision is required before anything proceeds.

Expected weekly deposit not received after 3 days → immediate alert.
Integrations

Everything the business runs on — connected.

FCore Agent integrates with tools already in use. No migration, no new software to learn.

📊
Accounting core
QuickBooks Online

Read & write. Invoices, payments, journal entries, Chart of Accounts, reconciliation. The single source of truth everything writes to.

🏦
Primary verification anchor
Bank Feeds via Plaid

Real-time transactions. Every QBO entry must reconcile to a bank record. The bank doesn't lie — it's the ground truth for the entire system.

🍞
Revenue source
POS — Toast / Square

Daily sales, payment types, refunds. Verified against bank deposits every day. Revenue is confirmed before it's recorded.

💼
Payroll verification flow
Gusto

Gross wages, taxes, net pay — cross-checked against actual bank disbursements. Payroll entries must resolve to real bank transactions.

✈️
Operator interface + documents
Telegram

The agent lives in existing chats. Receives files (invoices, statements, receipts), sends structured questions, takes operator decisions — where the team already works.

🏪
Counterparty reconciliation
Vendor Portals

Vendor statements parsed from PDF. Compared against QuickBooks AP balances to confirm that every obligation is captured, not just what's been entered.
